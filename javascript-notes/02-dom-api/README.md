## What is DOM?

> **DOM (Document Object Model)** = The browser's internal representation of a web page as a **tree of objects** that JavaScript can manipulate.

### One-Sentence Summary

> **DOM is the bridge between HTML (static content) and JavaScript (dynamic behavior).**

### Key Points

| Concept | Explanation |
|---------|-------------|
| **What it is** | A programming interface for web documents |
| **What it does** | Represents the page as objects/nodes in a tree structure |
| **Why it exists** | To allow JavaScript to read, modify, delete, or create HTML elements |
| **Analogy** | HTML = blueprint → DOM = robot with joints → JS = the operator |

### Without DOM vs With DOM

| Without DOM | With DOM |
|-------------|----------|
| Static image | Living document |
| No JS interaction | Full JS control |
| Page reload needed for changes | Real-time updates |

### Bottom Line

> **DOM turns HTML into an object tree that JavaScript can talk to and control.**
